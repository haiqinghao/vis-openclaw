## 系统概述

VIS OpenClaw 是一个用于监控和管理 OpenClaw 多 Agent 系统的可视化平台。

### 核心概念

| 概念 | 说明 |
|------|------|
| **Agent** | AI 智能助手，执行具体任务 |
| **Session** | 会话，与 Agent 的对话 |
| **Task** | 任务，分配给 Agent 的工作 |
| **Gateway** | OpenClaw 网关服务 |

### 系统架构

```
用户 → VIS OpenClaw 界面 → OpenClaw Gateway → Agent
```

## V1.1 更新亮点

V1.1 的核心目标是把 OpenClaw 的 Agent 协作过程从“列表和日志”升级为“可观察、可理解、可互动的任务现场”。新版新增游戏界面、地图编辑器、地图预览、单位演示和环境素材展示，让任务分发、Agent 响应和协作关系以更直观的方式呈现。

### OpenClaw × VIS 联动

任务从 VIS OpenClaw 分发后，会同步进入目标 Agent 的 OpenClaw 会话窗口，并在 VIS 游戏界面中形成可观察的任务现场。用户可以同时确认“消息是否进入 Agent 会话”和“游戏界面是否同步展示任务与 Agent 动作”。

![OpenClaw 与 VIS 联动](./usershouce/game/openclaw%26vis%20openclaw.png)

### 游戏界面

游戏界面把当前任务、参与 Agent、任务虚拟形象和地图环境放在一个可拖动、可缩放的画布中，适合做实时演示、任务验收和协作关系观察。

![游戏界面](./usershouce/game/youxijiemian.png)

### 地图编辑与预览

地图编辑器支持地图尺寸、缩放比例、画笔、橡皮擦、拖动、填充、清空、导入和导出。预览页用于检查已保存地图的实际展示效果，并选择哪张地图用于游戏界面。

![地图编辑器](./usershouce/game/mapbianji2.png)

![地图预览](./usershouce/game/mapyulan.png)

### 单位与环境素材

V1.1 接入了更丰富的 Tiny Swords 风格素材，用于虚拟 Agent、任务现场、地图地形、建筑和环境装饰。

![单位演示](./usershouce/game/danweiyanshi.png)

## 功能详解

### 1. 监测面板

监测面板是主界面，显示系统整体状态。

![监测面板](./usershouce/jiancemianban.png)

#### 状态指标

| 指标 | 说明 |
|------|------|
| Agent 数量 | 当前配置的 Agent 总数 |
| 会话数量 | 活跃会话数 |
| 任务数量 | 已创建的任务数 |
| 已分发任务 | 已分发给 Agent 的任务数 |

#### 任务管理

**创建任务**

1. 点击「创建任务」按钮
2. 填写任务名称和描述
3. 在描述中使用 `@agent名` 提及 Agent
4. 点击创建

**分发任务**

1. 找到待分发的任务卡片
2. 点击「分发」按钮
3. 系统将任务发送给相关 Agent

**定时任务**

1. 点击任务卡片的「定时」按钮
2. 设置定时模式：
   - **间隔启动**：每隔 N 分钟/小时启动
   - **定点启动**：每天固定时间启动
3. 保存配置

#### 任务状态

| 状态 | 说明 |
|------|------|
| 待分发 | 任务已创建，等待分发 |
| 已暂停 | 任务尚未分发，可恢复后再分发 |
| 定时 | 任务已设置定时 |
| 分发中 | 系统已接受分发请求，正在写入 Agent 会话 |
| 已分发 | 任务已发送给 Agent |
| 处理中 | Agent 会话正在处理任务 |
| 已完成 | 关联 Agent 会话已完成本轮处理 |
| 失败 | 任务分发或会话处理失败 |
| 状态待确认 | Agent 会话状态暂时无法确认，需要刷新或等待 Gateway 恢复 |

任务一旦进入「分发中」「已分发」或「处理中」，状态就由 OpenClaw 会话桥接链路接管，不再支持暂停；如果需要调整任务，请创建新任务或在 OpenClaw 会话侧处理。

### 2. Agent 管理

管理 OpenClaw 的智能助手。

![Agent 管理](./usershouce/agentguanli.png)

#### Agent 列表

显示所有 Agent 及其状态：

| 信息 | 说明 |
|------|------|
| 头像/Emoji | Agent 的标识 |
| 名称 | Agent 名称 |
| 模型 | 使用的 AI 模型 |
| 状态 | 在线/离线 |

#### 创建 Agent

1. 点击「创建 Agent」
2. 填写基本信息
3. 选择 AI 模型
4. 编辑配置文件
5. 保存

#### 编辑配置文件

每个 Agent 有 8 个配置文件：

| 文件 | 用途 |
|------|------|
| IDENTITY.md | 身份定义 |
| SOUL.md | 核心价值观 |
| USER.md | 用户/团队信息 |
| AGENTS.md | 工作流程 |
| MEMORY.md | 长期记忆 |
| TOOLS.md | 工具配置 |
| HEARTBEAT.md | 定期检查 |
| BOOTSTRAP.md | 首次启动 |

#### Agent 通信配置

配置 Agent 之间的通信权限：

1. 选择要配置的 Agent
2. 在「通信配置」中选择允许通信的 Agent
3. 保存配置

### 3. 会话管理

查看和管理所有会话。

![会话管理](./usershouce/huihuaguanli.png)

#### 会话列表

左侧显示所有会话：

| 信息 | 说明 |
|------|------|
| Agent 名称 | 所属 Agent |
| 类型 | 主会话/Subagent |
| 会话 ID | 会话唯一标识 |
| 消息数 | 会话消息数量 |

#### 消息区域

右侧显示选中会话的消息历史：

- 用户消息（蓝色）
- AI 回复（绿色）

#### 发送消息

1. 选择会话
2. 在底部输入框输入消息
3. 按 Ctrl+Enter 或点击「发送」

## 常见问题

### Q: 如何停止服务？

运行 `stop.bat` 或关闭命令行窗口。

### Q: 网关显示未连接？

1. 确认 OpenClaw 已安装
2. 点击「启动网关」按钮
3. 检查 Gateway 服务状态

### Q: Agent 显示离线？

1. 检查 Agent 配置是否正确
2. 确认 Gateway 服务运行中
3. 尝试重启 Gateway

### Q: 任务分发失败？

1. 确认目标 Agent 在线
2. 检查 Agent 通信配置
3. 查看 Agent 配置文件格式

### Q: 消息发送失败？

1. 确认会话是活跃状态
2. 检查后端服务是否正常
3. 查看浏览器控制台错误
